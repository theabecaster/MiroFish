## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, form field gap |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout gaps, form section separation |
| 2xl | 48px | Major section breaks, hero padding |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: Touch targets for the Google OAuth button and tab switcher must be minimum 44px height (accessibility standard for mobile tappability).

**Source:** Standard 8-point scale (default). Exception from WCAG 2.5.5 touch target guideline.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 24px | 700 (bold) | 1.2 |
| Display | 40px | 700 (bold) | 1.1 |

**Weight rationale:** Two weights only — 400 and 700. Label at 14px is visually distinct from Body at 16px through size alone; a third weight is unnecessary. Display at 40px reads as dominant from scale alone; extrabold (800) over bold (700) is imperceptible at that size in practice.

**Font pairing:**
- Headings and Display: `Plus Jakarta Sans` — rounded, energetic, marketers read this as confident and modern (source: CONTEXT.md D-04, RESEARCH.md recommendation)
- Body and Label: `Inter` — neutral, highly legible at small sizes, pairs well with Plus Jakarta Sans
- Both fonts loaded via `next/font/google` with `display: swap` and Latin subset

**Tailwind v4 configuration (in `globals.css`, not a config file):**
```css
@import "tailwindcss";

@theme {
  --font-heading: 'Plus Jakarta Sans', sans-serif;
  --font-body: 'Inter', sans-serif;
}
```

**Source:** RESEARCH.md Standard Stack note on `next/font/google`; CONTEXT.md D-04.

## Color

### Light Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAF9` (warm white, stone-50 equivalent) | Page backgrounds, auth card backgrounds |
| Secondary (30%) | `#F5F0EB` (warm off-white, sand-100) | Input fields, tab bar, nav bar, footer |
| Accent (10%) | `#F97316` (orange-500) | Primary CTA buttons, active tab indicator, link hover, focus rings |
| Destructive | `#EF4444` (red-500) | Error messages, form validation errors only |
| Text primary | `#1C1917` (stone-900) | All body text |
| Text secondary | `#78716C` (stone-500) | Placeholder text, helper text, labels |

### Dark Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1C1917` (stone-900) | Page backgrounds |
| Secondary (30%) | `#292524` (stone-800) | Input fields, tab bar, nav bar, cards |
| Accent (10%) | `#FB923C` (orange-400) | Primary CTA buttons, active tab indicator, link hover, focus rings |
| Destructive | `#F87171` (red-400) | Error messages, form validation errors only |
| Text primary | `#FAFAF9` (stone-50) | All body text |
| Text secondary | `#A8A29E` (stone-400) | Placeholder text, helper text, labels |

**Gradient (brand hero element):**
- Light: `from-orange-400 via-rose-400 to-purple-500` (applied to landing page hero heading and CTA button hover glow)
- Dark: same gradient, slightly reduced opacity (0.85)

**Accent reserved for:** Primary CTA button fills, active tab indicator underline, input focus rings, text links on hover — nothing else. Gradient used only on landing page hero and button hover effects.

**Rationale:** Warm stone base (not pure white/black) gives the "warm" feel from D-02. Orange accent is vibrant without clashing. Gradient used sparingly keeps the Canva/Loom boldness without overwhelming auth forms.

**Dark mode implementation:** Tailwind v4 `@custom-variant dark (&:where(.dark, .dark *))` in `globals.css`. `next-themes` controls the `.dark` class on `<html>`. System preference auto-detected on mount.

**Source:** CONTEXT.md D-02 ("bold & playful, vibrant gradients, warm colors"), D-03 (dark mode). Color values are Claude's discretion as stated in CONTEXT.md.

## Screens

### Landing Page (`/`)

Public. Accessible without auth (AUTH-05).

**Layout:** Full viewport hero, centered. Navbar at top (logo + Sign In link). Hero: gradient display heading, 2-line value prop body, CTA button. Footer at bottom.

**Tagline (display copy):** "Know before you spend."
**Value prop (body copy):** "Upload your ad, describe your audience, get a plain-English simulation of how they'll react — before you spend a dollar."
**Primary CTA:** "Start for free" (links to `/login?tab=signup`)

**Source:** CONTEXT.md D-10 ("hero with tagline, value prop, and login/signup CTA").

### Login Page (`/login`)

Single page, tab switcher for Sign In vs Sign Up (AUTH-01, AUTH-02).

**Layout:** Full viewport, vertically centered. AuthCard (Card with AuthTabs inside). No navbar.

**Tab 1 — Sign In:**
- Email field, password field
- "Sign In" primary button (full width)
- "Forgot password?" ghost link below button
- Divider ("or")
- GoogleButton (full width)

**Tab 2 — Sign Up:**
- Email field, password field, confirm password field
- "Create account" primary button (full width)
- Divider ("or")
- GoogleButton (full width)
- Helper text: "By signing up, you agree to our terms. Your LLM API key stays yours — we never charge for AI."

**Post-signup state (email confirmation pending):**
- Replace form with: confirmation message copy (see Copywriting below)
- No redirect until user clicks email link

**Source:** CONTEXT.md D-05, D-06, D-07.

### Forgot Password (`/login?forgot=true` or modal overlay)

Inline state change within /login — show ForgotPasswordForm instead of AuthTabs.

**Layout:** Same AuthCard, ForgotPasswordForm replaces AuthTabs. Back arrow link to return to login.

### Reset Password (`/auth/reset-password`)

**Layout:** Same AuthCard, ResetPasswordForm. User arrives here from email link.

### Dashboard Placeholder (`/dashboard`)

Protected (AUTH-05). Shown after login (D-11).

**Layout:** Navbar (authenticated variant) + centered content area. No sidebar.

**Content:** Preflight logo/wordmark, "Simulations coming soon" heading (Heading style), one-line body, muted status badge. No actions.

## Interaction States

### Form Inputs

| State | Visual |
|-------|--------|
| Default | Secondary bg, stone-300 border (light) / stone-600 border (dark) |
| Focus | Accent color ring (`ring-2 ring-orange-500`), no border color change |
| Error | Destructive border (`border-red-500`), red helper text below |
| Disabled | 50% opacity, no-cursor |

### Buttons

| State | Visual |
|-------|--------|
| Default (primary) | Accent fill, white text, rounded-xl |
| Hover (primary) | Gradient overlay + subtle shadow glow |
| Loading | Spinner replaces label text, button disabled |
| Default (outline) | Transparent fill, accent border, accent text |
| Hover (outline) | Accent bg at 10% opacity |

### Auth Flow States

| Trigger | UI Response |
|---------|------------|
| Submit sign-in form | Button enters loading state; error shown inline below password field on failure |
| Submit sign-up form | Button enters loading state; success shows confirmation screen (no redirect) |
| Click Google button | Button enters loading state, Supabase OAuth redirect fires |
| Click forgot password | Form swaps to ForgotPasswordForm (no page navigation) |
| Submit forgot password | Success message replaces form (see Copywriting) |
| Arrive at /auth/reset-password with valid token | ResetPasswordForm displayed |
| Submit new password | Loading state → success message → redirect to /login after 2s |

## Accessibility

- All inputs have explicit `<label>` elements (not placeholder-only)
- Error messages are linked to inputs via `aria-describedby`
- Focus order: top to bottom, no focus traps except modal overlays
- Google button has `aria-label="Sign in with Google"` (icon-only fallback)
- Color contrast: stone-900 on stone-50 = 17.6:1 (exceeds WCAG AA 4.5:1)
- Touch targets: minimum 44px height on all interactive elements (GoogleButton, ThemeToggle, tabs)
- ThemeToggle has `aria-label="Toggle dark mode"` and `title="Toggle dark mode"` for desktop tooltip visibility
